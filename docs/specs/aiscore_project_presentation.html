Background

왜 만들었나

🎵 현실의 문제

  • 교회 성가대·합창단은 악보(이미지)로 연습
  • 파트별 반주 트랙이 없으면 자기 성부만 듣기 어렵다
  • 피아노 없이 혼자 알토 파트를 익히려면?
  • 디지털 악보 데이터가 거의 없다 — 이미지만 있다

📦 보유 자산

  • 새찬송가 645곡 PNG 악보 이미지
  • 동일 곡 NWC → MusicXML 변환본 644개
  • 이미지 ↔ 구조화 데이터 쌍 = Ground Truth

💡 해결 아이디어

악보 이미지를 AI가 읽어

  1. SATB 음표 구조를 추출하고
  2. 성부별로 음성을 합성한 뒤
  3. 웹/앱에서 바로 재생

→ 상용 서비스 없음, 외부 API 전송 불가 조건
→ 로컬 추론 AI를 직접 만들어야 한다

🎯 목표

음표 인식 정확도 ≥ 95%  ·  로컬 추론  ·  웹·모바일 서비스

7
Technical Hurdle 3 — Most Critical

세 번째 허들: "학습이 되는 것처럼 보였다"

버그: T=8 사일런트 실패 
# 최초 설정
IMG_W = 256  # 작아서 빠른 테스트용
ResNet18 stride = 32
→ T = 256 / 32 = 8 프레임

# CTC 조건
T ≥ target_length 위반 시
zero_infinity=True → loss = 0.0

→ 모델이 아무것도 학습 안 했지만
  val_loss = 0.0000 으로 표시
  "완벽해 보이는" 숫자!

왜 발견이 늦었나

합성 데이터(시퀀스 길이 짧은)에서는 loss가 정상적으로 감소. 실제 찬송가 데이터로 바꾸자 val_loss = 0.0000 — 그제서야 이상 감지.

해결: stride 완화 + 입력 확대 
# Fix 1: maxpool 제거
backbone.maxpool = nn.Identity()
→ stride 32 → 16

# Fix 2: 입력 폭 확대
IMG_W = 256 → 2048
→ T = 2048 / 16 = 128 프레임

# Fix 3: EOS 토큰 제거
CTC target에 EOS 불필요
→ T 여유 확보

수정 후 학습 곡선

2 epoch 스모크: val_loss 6.59 → 6.12
실제 학습이 이루어지고 있음을 확인
